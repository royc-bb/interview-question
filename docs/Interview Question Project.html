General Information

Requirements

This project is required to deliver a stand-alone component that will:

  • provide a REST service to shorten any provided URL

  • provide a REST service to return the full URL when provided with the short version

  • provide persistent storage of the provided URL’s

  • periodically purge the URL’s based on age (currently older than 30 minutes)

Technical Constraints

The following technical constraints will direct the solution for this component:

  • Spring Boot must be used (a Spring Boot template application has been provided)

  • H2 in memory database must be supported

  • Lombok annotations for boilerplate code reduction must be used where applicable

Solution Description

A stand-alone Spring Boot application will be delivered and provide all required functionality within a single component. As this is a Phase 1 delivery this includes all internal components from the client-side REST interface to the underlying database.

Production Solution Notes

Security Risks

Although this is a simple URL shortening service the service will require protection from bad actors. If this system is compromised the URL’s could be altered to redirect clients to malicious websites.

Risk Mitigation

Having a single component may not be desirable for a production deployment from a security perspective. It’s not recommended to deploy this “Phase 1” component into a production environment as the database is not segmented from the client-side application.

For a production level deployment please review the following recommendations:

  • deploy a database in a secure network segment (or use database as a service ie. Amazon RDS)

  • separate the data services from this component and run those as a separate component in a secure network segment separate from the database

  • Push the client-side REST interface to the edge of the network and implement a caching solution to increase the response time for requested full length URL’s

Deployment Notes

The application.yml configuration file needs to be configured with the following details before the component can be successfully deployed:

  • server port (default: 5000)

  • H2 database file location and name (default: is the current working directory and named “interview”

  • H2 database username (default: sa)

  • H2 database password (default: password)

  • the amount of time a record should exist before being purged (default: 30 minutes)

  • the frequency of purging old records (default: 10 minutes)

    • note: purging too frequently may cause performance degradation (it locks the database for other write operations)

  • the minimum and maximum length for the short URL’s (default: 10-14)

    • note: the longer the maximum value and the wider the gap between min and max provides for significantly more short URL’s available

  • REST endpoint and query parameter names for requesting a short URL (default: /short?url)

  • REST endpoint and query parameter names for requesting a full lenth URL (default: /long?tiny)

  • error message text to return when the requested short URL is not found (default: “URL not found in DB”)
